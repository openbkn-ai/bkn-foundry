🔑 认证 —— 获取并携带 Token

下列接口默认需要认证。请求时在 HTTP 头带上 Authorization: Bearer <token>。获取 token 有两种方式：

  1. CLI 登录（推荐）：用 bkn-sdk 的 openbkn CLI 登录，token 存到 ~/.bkn，CLI 自动携带并刷新： 
    openbkn auth login https://<your-instance> -u <user> -p <password> -k
# 之后直接调用，如：openbkn bkn list
  2. OAuth2 client credentials：服务间调用用 POST /oauth/token（clientCredentials 流）换取 token；或使用应用密钥 AppKey（bak_ 前缀）以 Authorization: Bearer bak_xxx 直接携带。

拿到 token 后，可直接调用接口：

curl -H "Authorization: Bearer <token>" https://<your-instance>/api/bkn-backend/v1/...
