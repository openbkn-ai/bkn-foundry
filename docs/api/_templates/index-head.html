🔑 认证 —— 获取并携带 Token

下列接口需要认证。请求时在 HTTP 头带上 Authorization: Bearer <token>。拿到 token 后即可调用：

curl -H "Authorization: Bearer <token>" \
     https://<your-instance>/api/bkn-backend/v1/knowledge-networks

方式一 · CLI 登录（交互 / 人用，推荐）

用 bkn-sdk 的 openbkn CLI 登录一次，token 存到 ~/.bkn/，之后所有命令自动携带并按需刷新。登录走 OAuth2 设备码流（RFC 8628）：

# 浏览器登录（打印 URL + user code，在浏览器里确认）
openbkn auth login https://<your-instance>

# 无浏览器 / CI：用户名密码，CLI 在后台代走 login/consent 页
openbkn auth login https://<your-instance> -u <account> -p <password> -k

# 登录后直接调用
openbkn bkn list

方式二 · AppKey（程序 / 自动化，推荐）

AppKey（bak_ 前缀）是长期有效的持有者凭据，以签发人身份调用接口。先用已登录的会话签发一枚（AppKey 不能再签发 AppKey）：

# 签发：需已登录（OAuth token）。name 必填；不传 expires_at 默认 1 年
curl -X POST https://<your-instance>/api/safe/v1/me/api-keys \
  -H "Authorization: Bearer <your-oauth-token>" \
  -H "Content-Type: application/json" \
  -d '{"name":"my-key","expires_at":"2027-01-01T00:00:00Z"}'

# 201 响应，明文 key 只在此出现一次，务必保存：
# {"id":"...","key_id":"2882bb603277","name":"my-key",
#  "key":"bak_2882bb603277_BSxTZHTBiJXk4QoUBc0u17zSWua",
#  "masked":"bak_2882****SWua","expires_at":"...","enabled":true}

# 用 AppKey 调用（同样是 Bearer）：
curl -H "Authorization: Bearer bak_2882bb603277_BSxTZHTBiJXk4QoUBc0u17zSWua" \
     https://<your-instance>/api/agent-retrieval/v1/kn/search_schema

CLI 等价：openbkn appkey create --name my-key；调用时 openbkn --token bak_... ...。管理：GET /api/safe/v1/me/api-keys 列出、DELETE /api/safe/v1/me/api-keys/{id} 撤销、POST /api/safe/v1/me/api-keys/{id}/regenerate 轮换。

方式三 · OAuth2 直接换 token（协议细节）

token 端点是 Hydra 的 POST /oauth2/token（不是 /oauth/token）。服务间调用用 client credentials（需预先配置了 client secret 的客户端，如 ci-runner；凭据放请求体，client_secret_post）：

curl -X POST https://<your-instance>/oauth2/token \
  -H "Content-Type: application/x-www-form-urlencoded" \
  --data-urlencode grant_type=client_credentials \
  --data-urlencode client_id=<client-id> \
  --data-urlencode client_secret=<client-secret> \
  --data-urlencode audience=bkn-safe
# -> {"access_token":"...","token_type":"bearer","expires_in":...}

交互式登录（CLI）用的是设备码流：POST /oauth2/device/auth 取 device code，再轮询 POST /oauth2/token（grant_type=urn:ietf:params:oauth:grant-type:device_code）。绝大多数场景用方式一或方式二即可，无需手写本流程。
